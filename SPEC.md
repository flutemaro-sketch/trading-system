# 日本株 前日検討システム 仕様書

## システム概要
らいおんまるさんの手法に基づいた前日検討スクリーニングシステム。
自分で登録した銘柄リスト100銘柄をJ-Quants APIでデータ取得し、
以下の条件でスコアリングして翌日候補銘柄を画面表示する。

## 基本概念
- AIスコア = リスクリワードの数値化
- 全ての事案を数値化して理解することが前提
- 大口の動きを先読みして待ち受けることが目的
- 個人が負けて聖地完了後に上昇する
- AIの優位性が高い = 大口が動ける環境が整っている

## スコアリング条件

### ① 下駄パターン(20点)
- 上髭が実体の4倍以上(1:4の比率)
- 寄引同値または近似値(±0.3%以内)
- 出来高が前日比150%以上
- 翌日続き足になりやすい特性

### ② 出来高条件(20点)
- 出来高が5日平均の2倍以上で急増
- 出来高だまりの形成確認
- 500万株以上の出来高許容量
- VWAP・TWAPの確認
- 価格帯別出来高(出来高だまり)の検出
- ひげ部分は出来高0換算

### ③ トレンド条件(20点)
753スコアの計算:
- n=7(直近7日)×15
- n=5(直近5日)×21
- n=3(直近3日)×35
- 陽線○と陰線●をカウントして重み付き集計
- 陽線優勢ならスコア加点

強いトレンドの条件:
- 底値切上げ
- 高値更新
- 陽線主導
- 5日・25日・75日移動平均が上向き
- 山→BOX→山パターン(上昇継続)
- 一度踊り場・横横をはさみながら上昇が強い

### ④ 信用残条件(20点)
- 信用買残が増加傾向(新陳代謝進行)
- 信用売残が増加 = 踏み上げのガソリン
- 取組倍率の確認
- 高値圏の信用買残増加は悪材料(減点)
- 安値圏の信用買残増加は好材料(加点)
- 高値で掴んだ信用買残が減って、安値で掴んだ信用買残が増えている = 新陳代謝完了
- 貸株残新規大量発生+大陽線形成は加点
- 信用残は週1回火曜日更新・毎日引けに更新

### ⑤ アイランド・パターン条件(20点)
- アイランド形成の確認(出来高だまりを使ったAIラインでブロックが分かれた領域)
- GU後の出来高こなし確認
- 続き足パターン(前日終値からほぼ同値で寄付)
- お盆型パターン(アイランド形成後にくるんとゆるやかに戻ってくる)
- 増担解除銘柄の検出(解除後AIスコアが上がる)
- グループA銘柄の確認(材料で一気に急騰した銘柄・出来高許容量が増えた)
- 緩斜化の条件確認(急激な下落+大陰線+出来高許容量が多い)

## エントリー判断基準の表示
各銘柄に以下のコメントを自動表示:
- 翌日GUなら続き足候補
- GDは入らない
- 小GU(1%前後)が理想
- 寄引同値なら高確率(80%以上)
- 増担解除銘柄は要注目
- アイランド形成済みなら侵食回避を狙う

## 重要セクター判定
以下のセクターに属する銘柄は加点:
- 生成AI・デジタルセンター関連
- 軍事・造船(国防・日本船舶協力)
- 宇宙開発・ドローン(国防・防災対応)
- 半導体・仮想通貨
