# 板監視システム（kabuStation リアルタイム監視）

## プロジェクト概要
kabuStation WebSocket API を使用して、複数銘柄の板情報をリアルタイムで監視するシステム。
実在の証券ツール（マネックス証券マルチボード500など）を参考に、複数銘柄が同時に見える効率的なUIを実装。

## 現在の実装状況（2026-05-14）

### ✅ 完成した機能
- **メインウィンドウ**: 上下2段×タブ5個で10銘柄同時表示
- **板パネル（v2改善版）**: 小さくコンパクトなレイアウト
  - ヘッダー: 銘柄コード + 現在値 + 騰落率
  - 情報行: 始値、高値、安値、出来高
  - 板: 売り5段 + 買い5段
  - 色分け: 売=赤系、買=緑系
  - フラッシュ: 注文変化時（売=ピンク、買=水色、300ms）
- **詳細ウィンドウ**: 左パネル（銘柄情報）+ 右パネル（板）
- **インタラクション**:
  - 左クリック: 詳細ウィンドウ表示
  - ダブルクリック: 銘柄登録ダイアログ
- **WebSocket連携**: kabuStation API からのリアルタイムデータ受信

## 重要な実装ポイント

### 🔴 レイアウト設計（expand=False が必須）
前回の失敗原因「expand=True による縦方向への拡大」を完全に排除：

```python
# ✅ 正しい（expand=False）
sec_frame.pack(fill=tk.X, expand=False)
row_frame.pack(fill=tk.X, expand=False)

# ❌ 間違い（expand=True で行が拡大して見える）
sec_frame.pack(fill=tk.BOTH, expand=True)
```

### 🎨 色分け設計
- **売り気配**: 背景 #330000（暗い赤）、テキスト #ff6666（明るい赤）
- **買い気配**: 背景 #003300（暗い緑）、テキスト #66ff66（明るい緑）
- **フラッシュ**: 売 #ffb6c1（ピンク）、買 #add8e6（水色）

## 技術仕様

### データフロー
```
kabuStation WebSocket
         ↓
websocket_client.py （Queue に蓄積）
         ↓
main_window._poll_queue() （データ取得）
         ↓
board_panel.update_data() （UI更新）
         ↓
[画面表示]
```

### スレッド構成
- **メインスレッド**: Tkinter GUI
- **データスレッド**: WebSocket 受信ループ
- **タイマースレッド**: フラッシュ効果（after）

### メモリ管理
- Queue ベースでスレッド間通信
- 銘柄登録 / 削除で動的にパネルを制御
- 詳細ウィンドウは複数開くことが可能

## 注意事項

### 必ず設定すること
1. `.env` ファイルに `KABU_API_PASSWORD` を設定
2. kabuStation API が localhost:18080 で起動していることを確認
3. Python 3.11以上を使用

### パフォーマンス
- フォントサイズ: 6pt（小さすぎる場合は 7pt に変更可能）
- 行高さ: 13px（固定）
- パネル幅: 140px（複数銘柄を並べるのに最適）
- 50銘柄同時表示時、CPU使用率は正常範囲内（テスト済み）

## トラブルシューティング

### レイアウトが崩れる場合
→ `expand=False` になっているか確認
→ ウィンドウサイズを手動で調整

### 板データが表示されない場合
→ `test_ui.py` でダミーデータが表示されるか確認
→ kabuStation API が起動しているか確認
→ `.env` の設定を確認

### フラッシュが見えない場合
→ 板データの更新頻度を確認
→ `_apply_flash()` の時間を延長（例: 500ms）
